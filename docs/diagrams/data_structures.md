# Data Structures Documentation

## ArrayList
**Stores:** List of exercises in the workout log.  
**Why Chosen:** Dynamic size, easy to add/remove elements.  
**Usage:** Stores user's exercise history.

## HashMap
**Stores:** Exercise categories and user preferences.  
**Why Chosen:** Fast lookup by key.  
**Usage:** Categorizes exercises and stores user settings.

## LinkedList
**Stores:** Sequence of exercises in workout plans.  
**Why Chosen:** Efficient insertions/deletions.  
**Usage:** Manages ordered workout plans.

## Stack
**Stores:** Exercises for undo operations.  
**Why Chosen:** LIFO structure for undoing last actions.  
**Usage:** Allows undoing recent additions.

## Queue
**Stores:** Pending workouts.  
**Why Chosen:** FIFO for scheduling.  
**Usage:** Manages upcoming exercises.
